# 序感科技官网内容策略 v1.0

## 1. 官网在前 12 个月承担什么

官网不是产品参数页，也不是融资材料的公开复制。它需要同时完成四件事：

1. 定义品类：解释 State Computing 为什么是下一代 AI 终端缺失的上下文层。
2. 建立可信度：公开序感的技术边界、隐私原则、不确定性治理和证据标准。
3. 承接合作：让终端厂商、设计伙伴、研究者与候选人快速判断是否值得进一步沟通。
4. 沉淀资产：持续发布洞察、研究进展、产品更新和岗位，形成可检索的长期内容库。

## 2. 对外定位

公司定位：序感科技是一家面向下一代 AI 终端的个人状态智能基础设施公司。

核心承诺：让每一个 AI 终端，理解此刻的你，也记得长期的你。

长期愿景：在用户授权下，让电脑、耳机、汽车、穿戴、机器人和空间设备共享最少且必要的状态上下文；让每个人拥有自己的状态记忆，而不是被平台拥有。

内容底线：愿景可以大，证据必须诚实；产品可以主动，用户必须拥有最终控制权。

## 3. 内容支柱

### A. State Computing

解释“状态”不是情绪标签，而是随时间变化、带置信度、可被纠正的认知负荷、疲劳、唤醒度、情绪效价与社交可用性。

### B. State Loop

展示产品最小闭环：信号 -> 状态估计 -> 长期记忆 -> 行动策略 -> 结果反馈。强调系统可以询问、延迟或保持安静。

### C. Trust Architecture

持续公开端侧优先、数据最小化、用途限制、可查看、可纠正、可撤销、可删除与可审计原则。

### D. Cross-terminal Memory

说明跨终端连续性的用户价值，但不把尚未完成的 State Passport 或开放标准写成已发布产品。

### E. Evidence Journal

记录真实用户验证、基线对比、失败假设、模型校准和产品迭代。所有数字必须带样本、周期、任务和状态标签。

- 管理后台：内容、媒体、线索、全局设置

## 6. 发布状态规范

- `Research`：研究方向或假设，尚未形成稳定交付。
- `Validating`：有原型，正在真实用户或合作终端中验证。
- `Building`：已进入明确产品开发，仍可能调整接口。
- `Available`：可被用户或合作伙伴实际使用。
- `Archived`：已停止投入，保留历史记录。

任何页面不得把目标准确率、目标用户数、预测收入、规划日期或合成数据测试写成已经达成的事实。

## 7. 一年内容节奏

- 每月：1 篇技术或产品深度文章，1 次进展更新。
- 每季度：1 份 State Computing 观察，公开假设变化与风险。
- 关键节点：设计伙伴、真实验证、跨端迁移、SDK 版本单独发布。
- 招聘：只发布已解锁岗位，岗位页写清责任、到位门槛与决策权限。
